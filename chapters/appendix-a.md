# 附录 A：推荐阅读路径

本书涵盖了 SGLang 从架构概览到底层实现的方方面面，不同背景的读者可以根据自身需求选择合适的阅读路径。

## 路径一：入门路径

**适合人群**：初次接触 LLM 推理引擎的开发者，希望快速上手 SGLang。

**推荐顺序**：

1. **第 1 章：SGLang 项目总览** — 了解项目全貌和设计理念
2. **第 18 章：OpenAI 兼容 API** — 掌握 API 使用方式
3. **第 2 章：服务器启动流程** — 理解启动参数和配置
4. **第 20 章：对比与选型指南** — 明确 SGLang 的定位和优势
5. **附录 B：核心参数速查** — 快速查阅关键配置

这条路径帮助读者在最短时间内建立对 SGLang 的整体认知，并能够独立完成基本的部署和使用。

## 路径二：进阶路径

**适合人群**：有一定 LLM 推理经验的工程师，希望深入理解 SGLang 的核心调度机制。

**推荐顺序**：

1. **第 3 章：请求的一生** — 端到端理解请求处理流程
2. **第 4 章：Scheduler 核心调度逻辑** — 深入批处理调度策略
3. **第 5 章：Continuous Batching 与 Chunked Prefill** — 掌握高效批处理技术
4. **第 7 章：Attention Backend** — 理解注意力计算优化
5. **第 6 章：RadixAttention 与前缀缓存** — 学习 SGLang 的核心创新
6. **第 8 章：KV Cache 内存池管理** — 了解内存管理策略
7. **第 9 章：模型加载与执行** — 理解模型运行时机制

这条路径覆盖了 SGLang 的核心调度和计算优化，适合希望进行性能调优或二次开发的工程师。

## 路径三：专家路径

**适合人群**：系统架构师或研究人员，希望全面掌握 SGLang 的高级特性和底层实现。

**推荐顺序**：

1. 完成路径二的所有章节
2. **第 6 章：RadixAttention 与前缀缓存** — 深入 Radix Tree 算法细节
3. **第 10-11 章：Tensor Parallelism / Expert Parallelism** — 掌握分布式推理架构
4. **第 12-13 章：结构化输出与 Jump Forward Decoding** — 理解约束解码的高级优化
5. **第 14 章：投机解码** — 学习推测执行加速技术
6. **第 15-16 章：量化支持与 CUDA Graph** — 掌握底层性能优化手段
7. **第 17 章：LoRA 动态加载** — 了解模型适配方案
8. **第 19 章：多模态支持** — 理解视觉语言模型的推理实现

这条路径适合希望对 SGLang 进行深度定制或为开源社区贡献代码的开发者。

## 灵活阅读建议

- 每章开头的引言和结尾的"本章小结"可以帮助快速判断该章是否值得深入阅读
- **附录 C：名词解释** 可随时查阅，帮助理解专业术语
- 各章之间的依赖关系较松，大部分章节可独立阅读
